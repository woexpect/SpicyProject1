SCRUMBoard

SCRUM Board es un aplicativo que le permite llevar el control de un proceso de desarrollo de software de inicio a fin. La plataforma permite, en tiempo real, revisar el proceso del desarrollo de un proyecto mediante la visualización de cada uno de los integrantes y clientes mediante una interfaz personalizada para cada miembro específico; para definir el flujo de la aplicación se definen enseguida algunos roles y palabras clave:

BackLog: Es conocido como la colección de características o historias de usuario que harán del producto algo genial.

Sprint: Es el conjunto de elementos priorizados y cronometrados pertenecientes al BackLog que definen el ciclo de lanzamiento de una parte del producto.

Burndown Chart (BDC): Es una herramienta de visibilidad de proyectos propia de SCRUM que permite mediante mediciones diarias visualizar la cantidad de trabajo y estimado de liberación del producto.

Product Owner (PO): Se encarga de que las características correctas entren en el BackLog del producto, representando a los usuarios y clientes del producto.

SCRUM Master (SM): Se encarga de que el proyecto tenga un proceso suave y de que todos los miembros del equipo cuenten con las herramientas necesarias para realizar sus tareas, además organiza las reuniones y la planeación de la liberación del producto.

Equipo de trabajo: Consta del personal humano que hará realidad la realización del producto. El equipo de trabajo está conformado por desarrolladores, testers, clientes y ejecutivos.
